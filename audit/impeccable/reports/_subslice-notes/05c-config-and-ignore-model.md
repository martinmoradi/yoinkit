## Deep Implementation Notes

- `DEFAULT_CONFIG` is accurately reported. `skill/scripts/hook-lib.mjs:72-81` includes `enabled`, `quiet`, `auditLog`, `designSystem`, the three ignore arrays, and `limits`. `cloneDefaultConfig` at lines 165-174 returns fresh arrays and fresh nested objects.
- `safeReadJson` is a true fail-open reader: it catches every read/parse failure and returns `null` (`hook-lib.mjs:106-112`). `hookSection` and `detectorSection` then reject non-object values (`151-159`), so a bad config file falls back to defaults rather than breaking the hook.
- The returned runtime config is flat. `readConfig` starts from the clone, applies `hookSection` and `detectorSection` for each file, and returns the same object (`hook-lib.mjs:137-148`). `runHook` reads `config.enabled` directly (`1316-1319`), checks `config.ignoreFiles` directly in the per-file loop (`1357-1360`), and passes the full config to `filterFindings` (`1391`).
- `designSystem.enabled` is not just schema trivia. `applyDetectorConfigSource` treats any value other than explicit `false` as enabled (`hook-lib.mjs:178-182`), and `designSystemOptions` returns `{}` when disabled (`1225-1234`). The CLI mirrors the gate with `detectionConfig.designSystem?.enabled !== false` in `cli/engine/cli/main.mjs:131-140`.
- Manual `npx impeccable detect` does share the detector ignore model. `cli/engine/cli/main.mjs` imports `readDetectionConfig`, `shouldIgnoreDetectionFile`, and `filterDetectionFindings` from `../../lib/impeccable-config.mjs` at lines 8-12; directory scans filter ignored files at lines 198-199, single-file scans at 243, and all findings are filtered at 257. `--no-config` bypasses this at lines 131-134.
- `ignoreValues` behavior is accurately described. `normalizeIgnoreValueEntries` accepts singular `file`, folds it into `files`, keeps `reason` and `createdAt`, and drops entries without rule/value (`hook-lib.mjs:385-408`). `isIgnoredFindingValue` makes an unscoped wildcard a no-op and requires scoped wildcard entries to match file globs (`628-639`). `findingMatchesScopedIgnoreFile` tries the raw path and all suffixes (`641-653`).
- Only five rule ids yield an ignorable value: `overused-font`, `bounce-easing`, `design-system-font`, `design-system-color`, and `design-system-radius` (`hook-lib.mjs:655-666`; CLI mirror at `impeccable-config.mjs:486-497`).
- The `design-system-color` parser really is byte-identical across hook and CLI if the whole range is compared: `hook-lib.mjs:234-376` equals `cli/lib/impeccable-config.mjs:184-326`. It normalizes to `r,g,b,aByte` and supports hex, rgb/rgba, hsl/hsla, slash alpha, percent rgb, and hue units.
- The bundled detector copy claim is sound. `scripts/lib/utils.js:14-23` copies `cli/lib/impeccable-config.mjs` to `lib/impeccable-config.mjs` for the detector bundle because `cli/engine/cli/main.mjs:8-12` imports `../../lib/impeccable-config.mjs`. A `diff -q cli/lib/impeccable-config.mjs .claude/skills/impeccable/scripts/lib/impeccable-config.mjs` returns exit 0.
- The hook runtime does not import `impeccable-config`. `rg` found no such import in `skill/scripts/hook-lib.mjs`, `hook.mjs`, or `hook-before-edit.mjs`; only `cli/engine/cli/main.mjs`, `cli/bin/commands/skills.mjs`, and `cli/bin/commands/ignores.mjs` import `cli/lib/impeccable-config.mjs`.
- The hook exclude writer is the robust one. `ensureHookGitExcludes` writes three patterns (`hook.cache.json`, `hook.pending.json`, `config.local.json`) from `HOOK_LOCAL_IGNORE_PATTERNS` (`hook-lib.mjs:83-87`), uses marker suffixes for subdirectory prefixes (`470-472`), walks up to find `.git` (`501-519`), handles `.git` files for worktrees/submodules (`521-530`), and prefixes patterns when `cwd` is below the git root (`467-469`, `509-512`).
- The CLI config exclude writer is narrower. `ensureConfigGitExclude` writes only `.impeccable/config.local.json` (`cli/lib/impeccable-config.mjs:585-600`), looks only for `.git` directly under `root` (`619-634`), and has no subdirectory pattern prefix.

It appears in `HOOK_LOCAL_IGNORE_PATTERNS` (`hook-lib.mjs:83-87`), `getPendingPath` (`126-128`), `hook-admin.mjs` reset deletion (`595`), `LIVE_IGNORE_PATTERNS` (`live-inject.mjs:33-40`), tests, and smoke evidence reads. ## Paradigms Worth Importing

- Steal the two-tier config shape, but make the merge contract explicit before YoinkIt copies it. Decide which fields are scalar override, additive union, duplicate-key replacement, or removable. Impeccable's current behavior is not "local fully overrides shared"; it is "local can add and override some keys."
- Steal the semantic value comparator idea. Impeccable's color suppression compares parsed RGBA, not literal CSS strings. YoinkIt should put easing, duration, transform delta, and viewport-tolerance comparisons behind canonical comparators so suppressions survive capture/authored format differences.
- Steal `.git/info/exclude` for machine-local state, not `.gitignore`. The source shows why it works for local consent/cache without touching team-visible files. But YoinkIt should centralize exclude writing, since Impeccable now has multiple writers with overlapping patterns.
- If YoinkIt splits CLI-published code from bundled skill code, prefer a generated or shared config module. Impeccable's duplicated color parser stays in sync today, but the glob matcher drift shows how small differences appear even in a documented copy.
- Treat consent as its own domain operation. Impeccable has both installer consent and admin `hooks on` consent writes. YoinkIt can avoid ambiguity by routing all consent writes through one function and by naming whether the action is "install prompt decision" or "runtime enable decision."
