## Deep Implementation Notes

- Confirmed selector filter: `isLikelyHashedClass` at `cli/engine/browser/injected/index.mjs:491-497` treats empty class names, `css-`/`sc-`/`emotion-`/`jsx-`/`module-` prefixes, underscore module names, and alnum+digit hashes as unstable.
- Confirmed segment construction: `buildSelectorSegment` at `injected/index.mjs:499-528` emits the lower-case tag, appends at most two non-`impeccable-`, non-hashed classes via `CSS.escape`, then uses `parent.querySelectorAll(':scope > ' + sel)` to decide whether `:nth-of-type(n)` is needed. If the scoped selector throws, it falls back to `${tag}:nth-child(idx)`, dropping classes for that segment.
- Confirmed full selector walk: `generateSelector` at `injected/index.mjs:530-563` returns `body`, `html`, or bare `#id` for the target element; otherwise it walks upward with `MAX_DEPTH = 10`, child combinators, and early return when `document.querySelectorAll(trySelector)` returns exactly the target.
- Confirmed direct text measurement: `getDirectTextRect` at `injected/index.mjs:572-598` creates a `Range` for each direct text node, collects non-trivial client rects, and returns their union with `left/top/right/bottom/width/height/x/y`. `analyzeVisualContrastCandidate` then uses `getDirectTextRect(el) || el.getBoundingClientRect()` at `injected/index.mjs:1114`, so direct glyph bounds are preferred but element bounds remain the fallback.
- Confirmed selector round-trip for visual contrast: `collectVisualContrastCandidates` stores `selector: generateSelector(el)` at `injected/index.mjs:696`. `analyzeVisualContrastCandidate` re-resolves `candidate.selector` at `1090-1095`, returning `stale selector`, `missing element`, or `hidden element` instead of measuring the wrong node. `addVisualContrastResult` re-resolves at `1624-1629` before decorating. `captureVisualContrastCandidate` re-resolves in Puppeteer at `screenshot-contrast.mjs:118-135` before applying the screenshot mask.
- Confirmed serialized finding shape: `serializeFindings` at `injected/index.mjs:1212-1233` emits `selector`, `tagName`, `rect`, `isPageLevel`, `isHidden`, and normalized finding metadata: `type`, `category`, `severity`, `detail`, `ignoreValue`, `name`, `description`.
- Confirmed extension-mode handshake: in `EXTENSION_MODE`, the engine installs a `message` listener for `source:'impeccable-command'` and actions `scan`, `toggle-overlays`, `remove`, `highlight`, and `unhighlight` at `injected/index.mjs:1855-1911`, then posts `{source:'impeccable-ready'}` at `1912`. Non-extension mode schedules `scan()` after DOM ready unless `autoScan === false` at `1913-1928`.
- Confirmed live-mode stronger locator: `documentRefForElement` builds a full structural ref at `skill/scripts/live-browser.js:3474-3488`; `documentRefClassSuffix` strips `impeccable-` classes at `3500-3508`; `buildPickedAnchorSnapshot` stores tag/id/classes/text at `4859-4866`; `findLiveElementFromAnchorSnapshot` does id-first, class-subset, text-needle resolution at `4898-4919`; and `resolveLiveInjectionAnchor` composes still-live element, snapshot lookup, original-markup lookup, and a class-overlap fallback at `4961-4983`.

## Paradigms Worth Importing

- Import the detector's CSS selector generator into YoinkIt's `pick()` as a `stableCssSelector` field, but do not make it the only identity. Pair it with the live-mode pattern: a structural ref plus `{tag,id,classes,text}` snapshot, re-resolved id-first before `on(sel)`, trigger, and `dump()`.
- Treat "stale selector" as a first-class capture result. Impeccable never silently measures a different element after a selector stops resolving; YoinkIt should surface the same state when a selected layer disappears between arm, trigger, and dump.
- Scrub tool footprint at every boundary: `scan(root)`, candidate creation, background/pixel sampling, raw HTML serialization, and final spec emission. The Impeccable pattern is not just "hide overlays from findings"; it prevents injected UI from becoming part of the measurement substrate.
- Use `getDirectTextRect` as the narrow text-layer measurement primitive for motion specs. For animated labels, marquees, split text, and button text, range-union bounds will produce better per-frame deltas than container rectangles.
- Copy the extension ready handshake shape: page-world engine exports a namespaced API, announces ready once loaded, then waits for commands. This directly addresses YoinkIt's "arm before `__cap` exists" race.

## Open Questions

- Should YoinkIt's public API continue to accept a plain selector in `on(sel)`, or should `pick()` return a richer locator object while `on()` accepts both? The audit evidence favors a richer internal locator even if the public surface keeps a selector string for convenience.
- Which non-YoinkIt injected surfaces should be scrubbed by default? Impeccable explicitly handles `claude-` and `cic-`; YoinkIt likely needs its own allow/deny list for `agent-browser`, Claude-in-Chrome, browser-extension overlays, and any future YoinkIt toolbar IDs/classes.
- How much of the hashed-class filter should YoinkIt copy verbatim? The current heuristic is useful, but YoinkIt may need fixtures for Tailwind utility classes, CSS Modules, Webflow-generated classes, and animation-library wrappers so stable utility classes are not discarded accidentally.
