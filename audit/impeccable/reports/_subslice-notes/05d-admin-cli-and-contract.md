## Paradigms Worth Importing

- Preserve the report's headline pattern for YoinkIt: teach the model the exact suppression command in the finding output, but make the only writer a fail-closed admin command that requires explicit user confirmation. This maps cleanly to capture coverage exceptions such as `ignore-element`, `ignore-interaction`, and `ignore-viewport`.
- Copy the two-posture split. A YoinkIt hook or coverage checker should fail open inside the agent turn, but `/yoinkit capture` admin writes should fail closed and loudly. The implementation evidence here is concrete: runtime hook tests cover swallowed detector failures, while `hook-admin.mjs` exits 1 on malformed admin input.
- Copy the broad-suppression guard as code, not just prompt text. Impeccable hard-rejects `ignore-rule overused-font` unless `--all-values` is explicit. YoinkIt should similarly reject broad capture-gate suppression when a narrow selector or interaction exception would preserve the useful guarantee.
- Copy the manifest-repair idempotence pattern, but generate or share the manifest shape if possible. Impeccable's runtime repair and build-time emitters are currently hand-maintained parallel definitions; that works because tests cover drift, but YoinkIt can avoid the drift surface by centralizing the shape.
