## Paradigms Worth Importing

- **Long-poll plus lease as a human-in-browser handoff primitive.** The key implementation pattern is not just "poll"; it is "park response, lease event, only remove on ack, reflush on lease expiry." That is the part YoinkIt can lift for real-browser capture sessions without depending on a specific agent harness.
- **Prelude-injected config for arbitrary page scripts.** Impeccable solves "raw injected script cannot import runtime config" by serving the script through a tiny local helper that prepends globals. YoinkIt can use the same trick for `capture-animation.js`: prepend collector URL, token, and any capture vocabulary.
- **Transport with separate trust channels.** The browser event path is durable-first and validation-gated; the normal agent reply path is best-effort for journaling because the browser still needs an SSE reply; manual apply is yet another early-return route. This split is more useful than a single abstract "message bus" model.
- **Presence should be visible and cheap.** `agent_polling` is an edge-triggered derived boolean, not a heartbeat protocol. The browser also periodically checks `/status` for the same boolean (`live-browser.js:9297-9309`). YoinkIt should show "agent listening" before the user invests in picking or triggering a capture.
- **Source-owning mechanics should not transfer blindly.** `/source`, completion vocabulary, carbonize reply shape, and HMR assumptions all depend on Impeccable owning the user's repo. YoinkIt should copy the transport skeleton and state safety, not the source-serving or source-writing semantics.
