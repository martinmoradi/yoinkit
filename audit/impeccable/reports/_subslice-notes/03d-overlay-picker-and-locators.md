## Paradigms Worth Importing

- Import `own()` plus 20x20 `pickable()` as a cheap, low-maintenance pick gate for YoinkIt. It is small enough to transplant directly, with only the chrome prefix changed.
- Import id-first re-resolution, but do not overfit the regex. YoinkIt should either store class confidence separately or deliberately ignore likely build-hash classes if it wants hash drift tolerance. The current Impeccable regex only drops non-identifier tokens.
- Import the explicit mount-root seam: a dependency-free engine reads a global root and falls back to body; framework adapters set the root before loading the engine. Keep shadow DOM opt-in, because the SvelteKit path shows the real extra cost of proxy hit targets.
- Import reversible footprint scrubbing as a contract: YoinkIt's `scan()` and `dump()` should make it impossible for toolbar/runtime attributes to appear in the emitted spec.
- Import scroll-lock as part of any workflow that mutates or reloads the page between pick and finalization. It matters for animation capture because selector re-resolution and viewport stability are coupled in a real browser.
- Import the "two adjacent mechanisms become one stronger YoinkIt locator" lesson: Impeccable uses picked snapshots for variant recovery and structural refs for manual-edit recovery. YoinkIt can intentionally store both in `on(sel)` or `pick()` output, but that is an adaptation, not a verbatim copy of the generate payload.
- Consider importing the reversible CSP meta patcher if YoinkIt serves a local engine script. It is concrete, idempotent, and cleans up after itself using `data-impeccable-csp-original`.
