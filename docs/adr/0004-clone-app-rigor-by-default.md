# Borrow clone-app rigor by default, relaxing only for YoinkIt's product

YoinkIt shares clone-app's cloning ambition, but not its source/code fidelity
shortcut. It adopts clone-app's procedural rigor by default: fixed artifacts,
evidence rules, binary gates, coverage manifests, `null + reason` for unknowns,
hard check-ins, and measured convergence. This is worth recording because future
readers may see clone-app-style strictness as contradictory to YoinkIt's clean
re-expression goal; the intended split is strict evidence and Static Fidelity,
paired with no source-code, source-framework, or source-architecture fidelity,
plus a confidence-aware motion gate.
