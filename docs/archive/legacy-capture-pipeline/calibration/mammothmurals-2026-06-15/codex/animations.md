## Needs verify
- **hero-load-split-lines** · Structural map found 6 heading split nodes and 8 paragraph split nodes. Load motion could not be live-armed after navigation.
- **navbar-scroll-state** · Callback string moves nav link group by yPercent -200, wordmark xPercent -150, CTA xPercent 150 when scrolling down past top -300. Timeline captured the wordmark x shift.
- **partners-logo-reveal** · ScrollTrigger callbacks call play/pause on a closed-over timeline. Capture showed eight logo target transforms clearing or settling; exact source timeline vars are not exposed.
- **work-cover-slideshow-crossfade** · Paired image layer moves inversely: second layer opacity 0 -> 1. Callback starts setInterval(H, 300) on enter and clears it on leave.
- **cta-collection-reveal** · Callback read: gsap.to(G,{scale:1,autoAlpha:1,filter:"blur(0px)",duration:I,ease:"ease-secondary",stagger:{each:u,from:"center"}}). Fresh computed state before scroll had opacity 0, hidden visibility, blur(5px).
- **nav-link-hover-overlap** · Primary label simultaneously scales 1 -> 0. 3D rotation values are matrix decompositions.
- **primary-button-hover-overlap** · Primary container simultaneously scales 1 -> 0. Same mechanism applies to .g_btn_main variants.
- **wordmark-character-hover** · Same stagger-char mechanism appears in menu link text and small g_link_span elements, but those extra hover candidates were skipped as duplicates or hidden menu contexts at this viewport.
- **faq-accordion-toggle** · Button close sample: div.accordion_css_state_close.u-text-style-mono y 0 -> 21px over 0.5s. CSS map also found icon SVG transform 0.6s ease-primary.
- **video-lightbox-control-hover-timings** · Timing read from static CSS hover sweep. The lightbox was not opened, so from/to deltas need visual verification.

## Can't spec-capture
- **remaining-hover-candidates** · Static map found 74 hover candidates. Captured representative nav, primary button, wordmark, work cover, work/view CTA, services CTA, and FAQ interactions; skipped descendants and duplicates sharing those mechanisms.
- **mobile-menu-interactions** · button.navbar_menu_btn was 0x0 and hidden at the required 1280x800 viewport, so desktop run did not capture mobile menu open/close motion.
- **video-lightbox-open-playback** · Showreel/lightbox was not opened. CSS hover timings for visible lightbox controls were mapped, but the modal open/close animation and Vimeo playback controls need a dedicated pass if required.
